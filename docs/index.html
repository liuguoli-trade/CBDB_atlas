CBDB Atlas · 部署說明

本頁說明為何 GitHub Pages 無法提供與 start_cbdb_atlas.bat 相同的完整功能， 以及如何在本地或雲端正確使用本應用。

重要：GitHub Pages 只能託管靜態 HTML/CSS/JS，無法執行 Python、FastAPI 或 SQLite。 因此 https://liuguoli-trade.github.io/CBDB_atlas/ 不能替代 雙擊 start_cbdb_atlas.bat 後開啟的完整界面與全部操作。

本地運行時的架構

雙擊 start_cbdb_atlas.bat 實際執行的是 python run.py，會在本機啟動：

瀏覽器 ←→ FastAPI 後端（Python） ←→ CBDB SQLite 資料庫 ↑ ↑ ↑ web/index.html /api/* 接口 cbdb.sqlite3 + 索引

人物檢索、傳記詳情、關係可視化、Excel 匯出、資料更新等所有功能都依賴後端 API 與本地資料庫。 僅把 web/ 目錄放到 GitHub Pages，API 請求會失敗，應用無法正常工作。

方案一：本地使用（推薦，與 start_cbdb_atlas 相同）

  1. 克隆：git clone https://github.com/liuguoli-trade/CBDB_atlas.git
  2. 安裝：pip install -e ".[dev]"
  3. 雙擊 start_cbdb_atlas.bat，或執行 python run.py
  4. 瀏覽器開啟 http://127.0.0.1:8770
  5. 首次使用在網頁點「更新資料」下載 CBDB，或手動放入 data/source/cbdb.sqlite3

這是項目設計的使用方式，支持全部功能。

方案二：雲端託管完整應用（公開 URL + 全部操作）

若需要一個公開網址、且支持與本地相同的全部操作，必須託管整個應用（前端 + Python 後端 + 資料庫）， 而不是 GitHub Pages。本倉庫已提供 Docker / Render 配置：

  1. 在 Render 新建 Web Service，連接本 GitHub 倉庫
  2. 選擇使用倉庫中的 render.yaml（Docker 部署）
  3. 部署完成後，在 Render 提供的 HTTPS 地址使用完整應用
  4. 首次啟動需在網頁中下載 CBDB 資料（體積較大，需足夠磁碟與記憶體）

注意：CBDB 資料庫體積大，免費雲端方案可能磁碟/記憶體不足；非商業研究用途須符合 CBDB 許可。
